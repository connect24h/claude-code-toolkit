# TDD Workflow スキル

テスト駆動開発のワークフローを自動的に適用します。

## トリガー

以下のキーワードが含まれる場合に自動適用:
- 実装
- 機能追加
- 新規開発
- バグ修正

## ワークフロー

```
1. SCAFFOLD: インターフェース/型を定義
       ↓
2. RED: 失敗するテストを書く
       ↓
3. 確認: テストが失敗することを確認
       ↓
4. GREEN: 最小限の実装
       ↓
5. 確認: テストが成功することを確認
       ↓
6. REFACTOR: コードを改善
       ↓
7. 確認: テストがまだ成功することを確認
       ↓
8. カバレッジ: 80%以上を確認
       ↓
9. REPEAT: 次の機能へ
```

## チェックリスト

- [ ] 型/インターフェースを定義した
- [ ] 失敗するテストを書いた
- [ ] テストが失敗することを確認した
- [ ] 最小限の実装をした
- [ ] テストが成功することを確認した
- [ ] リファクタリングした
- [ ] カバレッジ80%以上を確認した

## コマンド

```bash
# テスト実行
npm test

# カバレッジ確認
npm test -- --coverage

# 特定ファイルのテスト
npm test -- path/to/file.test.ts
```
