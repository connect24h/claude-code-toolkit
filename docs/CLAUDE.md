# CLAUDE.md - グローバル設定

Claude Code の動作を制御するグローバル設定ファイル

---

## 基本原則

### コア
- 全力で取り組む
- 思考は英語、応答は日本語
- 独立した操作は並列実行で効率化
- 複雑な問題はサブエージェントで検証
- ツール結果を受け取ったら、品質を評価し最適な次のステップを決定

### ワークフロー
- **Explore → Plan → Code → Commit** アプローチ
- 変更前に既存コードを読んで理解
- 実装前に詳細な計画を作成
- イテレーティブなアプローチ
- 早期かつ頻繁に軌道修正

### テスト駆動開発（TDD）
- **RED → GREEN → REFACTOR** サイクル
- テストを先に書く
- カバレッジ80%以上を目標

## オーケストレーション

Claude Codeは統括エージェントとして、Codex CLIとGemini CLIにタスクを委譲できます。

```
┌─────────────────────────────────────────────────────────┐
│                     vibe-kanban                         │
└─────────────────────────┬───────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    Claude Code                          │
│              （統括エージェント/オーケストレーター）      │
└───────────┬─────────────────────────────┬───────────────┘
            ↓                             ↓
┌───────────────────────┐     ┌───────────────────────────┐
│      Codex CLI        │     │       Gemini CLI          │
└───────────────────────┘     └───────────────────────────┘
```

### タスク振り分け

| 複雑度 | 担当 | タスク例 |
|--------|------|----------|
| 低 | Codex CLI | バグ修正、hotfix、1-2ファイル変更 |
| 中 | Gemini CLI | 分析、調査、ドキュメント生成 |
| 高 | Claude Code | 新機能、リファクタ、マルチステップ |

---

## 主要コマンド

| コマンド | 説明 |
|----------|------|
| `/orchestrate` | タスクを最適なCLIに振り分け |
| `/delegate-codex` | Codex CLIに委譲 |
| `/delegate-gemini` | Gemini CLIに委譲 |
| `/plan` | 実装計画を作成 |
| `/tdd` | TDDで実装 |
| `/code-review` | コードレビュー |
| `/verify` | 検証ループ実行 |
| `/build-fix` | ビルドエラー解決 |
| `/evolve` | パターンを進化 |

## ルール

### コーディングスタイル
- 関数は50行以内
- ネストは4レベル以内
- `any`型禁止
- `console.log`を本番コードに残さない

### セキュリティ
- 入力は必ず検証
- SQLはパラメータ化
- 機密情報はハードコードしない

### Git
- 意味のあるコミットメッセージ
- force push禁止
- 機密情報をコミットしない
