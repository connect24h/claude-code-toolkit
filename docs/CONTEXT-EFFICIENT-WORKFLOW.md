# コンテキスト効率化ワークフロー

Claude Codeのコンテキストを節約し、効率的に作業を進めるためのワークフロー設計。

---

## 基本原則

1. **計画ファースト** - 実装前に必ず計画を立てる
2. **分割統治** - 大きなタスクは小さく分割してサブエージェントへ
3. **コンテキスト分離** - サブエージェントは独自コンテキストを持つ
4. **適材適所** - タスク特性に応じてClaude/Codex/Geminiを使い分け

---

## ワークフロー図

```
┌─────────────────────────────────────────────────────────────────┐
│                    ユーザーリクエスト                           │
└─────────────────────────────┬───────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ フェーズ1: 計画（Plan）                                         │
│ ・要件を明確化                                                  │
│ ・成功基準を定義                                                │
│ ・大まかなステップを列挙                                        │
│ → 出力: 計画ファイル (.plan.md)                                 │
└─────────────────────────────┬───────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ フェーズ2: 詳細設計（Design）                                   │
│ ・影響ファイルを特定                                            │
│ ・依存関係を分析                                                │
│ ・タスクを独立した単位に分割                                    │
│ → 出力: タスクリスト（TaskCreate）                              │
└─────────────────────────────┬───────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ フェーズ3: 振り分け（Distribute）                               │
│                                                                 │
│  ┌─────────────┬─────────────┬─────────────┐                   │
│  │ 単純タスク  │ 分析タスク  │ 複雑タスク  │                   │
│  │ → Codex    │ → Gemini   │ → Subagent  │                   │
│  └─────────────┴─────────────┴─────────────┘                   │
│                                                                 │
│ ※ 各サブエージェントは独自コンテキストを持つ                    │
│ ※ 親エージェントのコンテキストは消費しない                      │
└─────────────────────────────┬───────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ フェーズ4: 実行（Execute）                                      │
│ ・並列実行可能なタスクは同時に実行                              │
│ ・各タスクの結果を収集                                          │
│ ・エラーがあれば対処                                            │
└─────────────────────────────┬───────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ フェーズ5: 統合・検証（Integrate）                              │
│ ・結果を統合                                                    │
│ ・テスト実行                                                    │
│ ・品質確認                                                      │
│ → 出力: 完了レポート                                            │
└─────────────────────────────────────────────────────────────────┘
```

## タスク振り分け基準

### Codex CLI へ委譲

```
条件:
- 1-2ファイルの変更
- 単純なバグ修正
- 明確な指示で完結
- 5分以内で完了

コマンド:
codex exec "タスクの説明"
```

### Gemini CLI へ委譲

```
条件:
- 大量のファイル読解が必要
- 分析・調査タスク
- ドキュメント生成
- 200万トークン級のコンテキストが必要

コマンド:
gemini "タスクの説明" --approval-mode yolo
```

### Task (Subagent) へ委譲

```
条件:
- 中程度の複雑さ
- 独立して完結可能
- コンテキスト分離が有効

使用:
Task tool with subagent_type
```

### Claude Code 本体で処理

```
条件:
- 複雑なマルチステップ
- 判断が必要
- 統合作業
- ユーザーとの対話が必要
```

## コンテキスト節約テクニック

### 1. サブエージェントの活用

```typescript
// ❌ 親コンテキストで全て処理
// → コンテキストが肥大化

// ✅ サブエージェントに委譲
Task({
  description: "ファイルX,Y,Zを分析",
  subagent_type: "Explore",
  prompt: "詳細な指示..."
})
// → サブエージェントは独自コンテキスト
// → 結果のみ親に返る
```

### 2. 計画ファイルの活用

```markdown
# 計画は必ずファイルに保存
/plan
→ .claude/plans/feature-x.plan.md に保存
→ 必要時に読み込み
→ コンテキストに常駐させない
```

### 3. 結果の要約

```typescript
// サブエージェントからの結果
{
  status: "success",
  summary: "3ファイルを修正",  // 要約のみ
  files_changed: ["a.ts", "b.ts", "c.ts"]
}
// 詳細はファイルに保存、コンテキストには要約のみ
```

### 4. 並列実行

```typescript
// 独立タスクは並列で実行
// → 各タスクが独自コンテキスト
// → 待ち時間削減

Task({ ... task1 ... })
Task({ ... task2 ... })
Task({ ... task3 ... })
// 3つ同時に実行
```

## 実践例

### 大規模リファクタリング

```
1. /plan で計画作成
   → plans/refactor-auth.plan.md に保存

2. 計画をTaskCreateでタスク分割
   - Task #1: 認証モジュール分析 → Gemini
   - Task #2: 型定義更新 → Codex
   - Task #3: テスト作成 → Subagent (tdd-guide)
   - Task #4: 実装変更 → Subagent
   - Task #5: 統合テスト → Codex

3. 並列実行可能なタスクを同時実行
   - Task #1, #2 を並列

4. 依存タスクは順次実行
   - Task #3 → Task #4 → Task #5

5. 結果統合・検証
```

## コマンドリファレンス

| コマンド | 用途 | コンテキスト影響 |
|----------|------|------------------|
| `/plan` | 計画作成 | ファイル保存（低） |
| `/orchestrate` | タスク振り分け | 自動委譲（低） |
| `/delegate-codex` | Codex委譲 | 分離（なし） |
| `/delegate-gemini` | Gemini委譲 | 分離（なし） |
| `Task tool` | サブエージェント | 分離（なし） |

## アンチパターン

### ❌ 避けるべき

```
1. 計画なしで大規模実装を開始
2. 全てを親コンテキストで処理
3. サブエージェントの結果を全て展開
4. 順次実行で時間浪費
5. 単純タスクにClaude Code本体を使用
```

### ✅ 推奨

```
1. 必ず計画フェーズから開始
2. タスク分割してサブエージェントへ
3. 結果は要約のみコンテキストに
4. 並列実行を活用
5. 適材適所でCLIを使い分け
```
