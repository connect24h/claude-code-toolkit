# /devils-advocate - 反対意見レビュー

Devil's Advocateエージェントを起動し、現在のプラン・設計・実装に対して意図的に反対の立場から批判的に検証する。

## 使い方

```
/devils-advocate                   # 現在のコンテキストを批判的に検証
/devils-advocate <ファイルパス>     # 特定のファイル/計画を検証
/devils-advocate <提案の説明>       # テキストで説明した提案を検証
```

## 実行内容

1. 対象のプラン・設計・提案を読み込む
2. Devil's Advocateエージェント（`agents/devils-advocate.md`）の観点で検証:
   - 暗黙の前提への疑問
   - 失敗モード分析
   - 見落としの発見
   - 代替案の提示
   - 隠れたコスト・リスクの特定
3. 危険信号を重要度別に分類（致命的/要注意/軽微）
4. 進行可否の判定（GO / 条件付きGO / STOP）

## パイプライン活用

```
/plan → /architect → /devils-advocate → 実装
```

設計評価後に反対意見レビューを挟むことで、見落としを防ぐ。

## 活用シーン

- 重要な技術選定の前
- 大規模リファクタリングの計画時
- 新規アーキテクチャの提案時
- 本番環境への変更前
- 意思決定で迷っている時
