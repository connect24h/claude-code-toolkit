# /architect - アーキテクチャ評価

Architectエージェントを起動し、現在の設計・計画・実装をアーキテクチャの観点から評価する。

## 使い方

```
/architect                    # 現在のコンテキストの設計を評価
/architect <ファイルパス>      # 特定のファイル/ディレクトリを評価
/architect <設計の説明>        # テキストで説明した設計を評価
```

## 実行内容

1. 対象のコード・設計・計画を読み込む
2. Architectエージェント（`agents/architect.md`）の評価観点で分析:
   - 構造的健全性（責務分離、依存関係）
   - 非機能要件（スケーラビリティ、保守性、セキュリティ）
   - 技術選定の妥当性
   - データ設計
3. 評価結果を構造化されたフォーマットで出力
4. 改善提案とトレードオフ分析を提示

## パイプライン活用

```
/plan → /architect → /devils-advocate → 実装
```

計画立案後にアーキテクチャ評価を挟むことで、設計品質を担保する。
