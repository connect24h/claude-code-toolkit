# /review-spec コマンド

仕様書を複数のAIでレビューし、ブラッシュアップします。

## 使用方法

```
/review-spec [仕様ファイルパス]
/review-spec  # 直前の計画をレビュー
```

## レビュープロセス

### ラウンド1: 並列レビュー（3つのAIが同時に）

| AI | 視点 | 確認ポイント |
|----|------|-------------|
| **Claude** | 設計 | アーキテクチャ、拡張性、セキュリティ |
| **Codex** | 実装 | 実装可能性、複雑度、テスト容易性 |
| **Gemini** | 分析 | 完全性、矛盾、代替案、リスク |

### ラウンド2: 統合

- 共通の指摘事項を抽出
- 矛盾点を解決
- 改善提案をマージ

### ラウンド3: 最終確認

- 改善後の仕様を再レビュー
- 全AIが承認したら完了

## 実行例

```bash
# 仕様ファイルをレビュー
/review-spec docs/api-spec.md

# 直前の計画をレビュー
/plan feature-x
/review-spec  # 作成された計画をレビュー
```

## 出力

```markdown

### Claude（設計視点）
- 👍 良い点: ...

### Codex（実装視点）
- 👍 良い点: ...
- ⚠️ 懸念: ...
- 💡 提案: ...

### Gemini（分析視点）
- 👍 良い点: ...
- ⚠️ 懸念: ...
- 💡 提案: ...

## ブラッシュアップ済み仕様
[改善後の仕様]
```

## ベストプラクティス

1. 重要な仕様は必ずレビューを通す
2. 矛盾するフィードバックは人間が判断
3. 承認前に全ての懸念点を解決
