# /tdd コマンド

tdd-guideエージェントを呼び出して、TDDで実装します。

## このコマンドがすること

1. **インターフェース定義** - 型/インターフェースを先に定義
2. **テストを先に書く** - 失敗するテストを作成（RED）
3. **最小限の実装** - テストを通す最小コード（GREEN）
4. **リファクタリング** - コードを改善（REFACTOR）
5. **カバレッジ確認** - 80%+を確保

## TDDサイクル

```
RED → GREEN → REFACTOR → REPEAT

RED:      失敗するテストを書く
GREEN:    テストを通す最小コードを書く
REFACTOR: コードを改善、テストは緑のまま
REPEAT:   次の機能へ
```

## いつ使うか

- 新機能の実装
- 新しい関数/コンポーネントの追加
- バグ修正（まずバグを再現するテスト）
- リファクタリング
- 重要なビジネスロジック

## 使用方法

```
/tdd [機能の説明]
```

## 例

```
/tdd ユーザーの年齢を計算する関数
```

tdd-guideエージェントが:
1. 型を定義
2. 失敗するテストを書く
3. テストが失敗することを確認
4. 実装
5. テストが成功することを確認
6. リファクタリング
7. カバレッジ確認

## カバレッジ要件

- **80%以上** 通常のコード
- **100%** 重要なコード:
  - 金融計算
  - 認証ロジック
  - セキュリティ関連
  - コアビジネスロジック

## 関連コマンド

- `/plan` - 先に計画を立てる
- `/verify` - 実装後に検証
- `/code-review` - 実装完了後にレビュー
