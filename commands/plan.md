# /plan コマンド

plannerエージェントを呼び出して、実装計画を作成します。

## このコマンドがすること

1. **要件の明確化** - 何を作るかを確認
2. **リスクの特定** - 潜在的な問題を洗い出し
3. **ステップ計画** - 実装を段階に分解
4. **確認を待つ** - ユーザーの承認を得る

## いつ使うか

- 新機能の開始時
- 大きなアーキテクチャ変更
- 複雑なリファクタリング
- 複数ファイルに影響する変更
- 要件が曖昧なとき

## 使用方法

```
/plan [機能の説明]
```

## 例

```
/plan ユーザー認証機能を追加したい
```

plannerエージェントが:
1. 要件を分析
2. 影響範囲を特定
3. 実装ステップを提案
4. リスクを評価
5. **確認を待つ**

## 重要

**コードは書かない** - 計画を承認するまでコードは書きません。

計画を変更したい場合:
- "修正: [変更内容]"
- "別のアプローチで"
- "フェーズ2を先に"

## 関連コマンド

- `/tdd` - 計画承認後に実装
- `/verify` - 実装後に検証
- `/code-review` - 実装完了後にレビュー
