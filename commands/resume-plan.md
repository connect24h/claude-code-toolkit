# /resume-plan - 計画再開

直近の Plan ファイルを読み込み、作業計画と進捗状況を把握する。
セッション再開時やコンテキスト圧縮後に、現在地を素早く復帰するためのコマンド。

## 実行手順

### 1. Plan ファイルの特定と読み込み

```bash
ls -t -1 ~/.claude/plans/*.md | head -n 1
```

上記で取得した最新の Plan ファイルを `Read` で全文読み込む。

### 2. 進捗状況の分析

Plan ファイルを読み込んだら、以下を分析・報告する:

#### a. 計画の概要
- プロジェクト名・目的
- 全体のフェーズ構成
- 技術スタック・制約事項

#### b. 進捗状況
- **完了済み**: チェック済み（`[x]`）の項目を一覧
- **進行中**: 部分的に完了しているフェーズ
- **未着手**: まだ手をつけていない項目
- **進捗率**: 完了項目数 / 全項目数（%表示）

#### c. 次のアクション
- 直近で取り組むべきタスクを特定
- 依存関係があれば明示
- ブロッカーがあれば警告

### 3. 関連ファイルの確認

Plan に関連するファイルの現在の状態も確認する:

- `tasks/todo.md`（存在すれば）の進捗
- `tasks/lessons.md`（存在すれば）の教訓
- 対象プロジェクトディレクトリの最新変更（`git log --oneline -5`）

### 4. 作業再開の提案

進捗分析に基づき、以下の形式で報告する:

```markdown
## 計画再開レポート

### 計画: [計画名]
### 進捗: [完了数]/[全数] ([%])

### 完了済み
- [x] ...

### 次のアクション
1. **[最優先タスク]**: [説明]
2. **[次のタスク]**: [説明]

### 注意事項
- [ブロッカー・依存関係・リスク]

### 推奨コマンド
- `[次に実行すべきコマンドの提案]`
```

## 使い方

```
/resume-plan                # 直近のPlanを読み込んで進捗確認
```

## 活用シーン

- セッション開始時（前回の続きから作業再開）
- コンテキスト圧縮後（`/compact` 後の現在地復帰）
- 長期プロジェクトの進捗確認
- 別のタスクから戻った時の状況把握

## 補足: Plan ファイルの進捗更新

実装のキリが良いタイミングで、Plan ファイル内のチェックボックスを更新すること:
- 完了: `- [ ]` → `- [x]`
- 完了日時や備考があれば末尾に追記: `- [x] タスク名 (2026-03-08 完了)`

## 関連コマンド

- `/plan` - 新規計画を作成
- `/architect` - 計画のアーキテクチャ評価
- `/devils-advocate` - 計画の批判的レビュー
- `/verify` - 実装後の検証
